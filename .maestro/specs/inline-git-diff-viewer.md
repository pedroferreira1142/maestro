# Inline git diff viewer

Let users review uncommitted changes without leaving Maestro. The Git panel (src/renderer/src/components/GitPanel.tsx) gains a list of changed files (staged, unstaged, untracked) under its status chips, backed by a new GitService function (src/main/GitService.ts) that parses `git status --porcelain=v2` into per-file entries, exposed through SessionManager (src/main/SessionManager.ts), ipc.ts, shared/api.ts and the preload bridge like the existing gitStatus/gitLog calls. Clicking a file opens a read-only diff tab in the session's tab strip: a new diff tab kind in the viewer state (ViewerState in src/renderer/src/store.ts uses plain relPath strings today, so diff tabs need a distinguishable encoding, e.g. a 'diff:' prefix) rendered by a new DiffViewer component modeled on FileViewer.tsx, displaying the unified output of `git diff HEAD -- <file>` in CodeMirror 6 with diff syntax highlighting (the existing @codemirror/language-data set includes a Diff language — no new dependency required). Diffs refresh on fsBus file-change events and gitNonce bumps, and the feature works identically in ordinary sessions and worktree task sessions, enabling pre-merge review of parallel tasks.

## Specs

- [ ] The Git panel of a session whose folder is a git repository lists each changed file (staged, unstaged/modified, and untracked) with its repo-relative path and a status indicator (e.g. M/A/D/U/?); the list updates after commits, merges, and manual refresh (gitNonce bumps), and is absent for non-repo folders.
- [ ] Clicking a changed file in the Git panel opens a diff tab in that session's tab strip, visually distinguishable from a plain file tab (e.g. a 'diff' marker in the label); clicking the same file again focuses the existing diff tab instead of opening a duplicate.
- [ ] The diff tab shows the unified diff of the file's working-tree state against HEAD (staged plus unstaged changes combined), rendered read-only in CodeMirror with added lines visually highlighted differently from removed lines and hunk headers.
- [ ] Opening the diff of an untracked file shows its full content presented as added lines rather than an empty diff or an error.
- [ ] An open diff tab reloads automatically when the underlying file changes on disk (the session's fs events) and when git state is refreshed (gitNonce), so the diff tracks Claude's ongoing edits live.
- [ ] Diff tabs behave like file tabs in the tab strip: they can be closed with their close control, the active-tab fallback still selects a terminal when the active diff tab is closed, and existing file tabs and terminals are unaffected.
- [ ] In a worktree task session, the Git panel file list and diff tabs operate on the worktree's own repository state, so a parallel task's changes can be reviewed in Maestro before clicking Merge.
- [ ] A binary changed file opens to a clear 'binary file' message instead of garbled text, and a very large diff is truncated with a visible notice rather than freezing the renderer.
