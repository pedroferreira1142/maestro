# Per-Session Environment Variables

Add an editable per-session key/value environment map that is merged into the environment of every terminal that session spawns (claude and shells alike). Persist it as a new optional `env?: Record<string, string>` field on `SessionConfig` in src/shared/types.ts, so it is saved automatically with the rest of session config in the app state file via Persistence. Thread the map from `SessionManager.spawnTerminal` (src/main/SessionManager.ts, the `new PtySession(terminal, config.folder, …)` call) into `PtySession` (src/main/PtySession.ts), where `spawn()` currently copies only `process.env` into the `env` object passed to `pty.spawn` (lines ~181-192) — overlay the session's entries on top of the inherited environment there. Add a setter on SessionManager (mirroring `setSessionCategory`, which returns the affected claude terminal ids for the renderer to restart) exposed over IPC (src/main/ipc.ts, src/preload/index.ts, src/shared/api.ts) and a store action (src/renderer/src/store.ts). Add a small editor dialog (a new component under src/renderer/src/components, opened from the session context menu in SessionSidebar.tsx) with add/edit/remove rows; on save, restart the session's affected running terminals via the existing `restartTerminal` plumbing (resume mode for claude) so the new environment takes effect.

## Specs

- [ ] A session context-menu entry (e.g. "Environment variables…") opens a dialog that lists the session's current variables as editable key/value rows; for a session that has never set any, the list starts empty.
- [ ] After adding variables and saving, then fully quitting and relaunching the app, the same variables reappear for that session (they are persisted on the session's config in the app state file).
- [ ] A shell terminal spawned in a session that defines VAR=hello shows `hello` when the variable is echoed (e.g. `echo $VAR` / `echo %VAR%`), confirming the value reached the spawned process environment.
- [ ] When a session variable has the same name as one inherited from the app's own environment, the spawned terminal sees the session's value (the per-session entry overrides the inherited one).
- [ ] Variables that the session does not define are still present in spawned terminals (the inherited environment such as PATH is preserved, not replaced).
- [ ] Saving a change to the variables (add, edit, or remove) automatically restarts that session's currently-running terminals so the new environment is in effect without the user manually restarting; terminals that were not running are not force-started.
- [ ] Two different sessions that each define the same variable name with different values each spawn terminals that see their own session's value (no cross-session leakage and no global mutation of the app environment).
- [ ] A variable row with an empty or whitespace-only key is not written into the spawn environment (it is ignored/rejected on save), and a session with an empty variable list spawns terminals with exactly the environment it had before the feature existed.
