# Stall & runaway watchdog for Claude sessions

Add a time-based watchdog that flags two abnormal conditions per Claude terminal that the instantaneous status badges miss: a 'working'/runaway terminal that has been continuously working longer than a stall threshold (likely stuck in a loop or a long no-op), and a 'needs-attention' terminal that has sat unanswered longer than an attention threshold (a permission prompt the user forgot). Implementation centers on src/main/SessionManager.ts (handleStatus is the single per-transition funnel — it already early-returns for non-claude terminals and already constructs new Notification(...) and calls win.flashFrame) plus src/main/StatusDetector.ts / src/main/PtySession.ts (add a 'time the current status was entered' stamp; today only lastOutputAt and current status exist). A main-side interval checks each claude terminal's elapsed time in its current status against the thresholds, raises a per-session watchdog flag surfaced on TerminalInfo/SessionInfo (src/shared/types.ts) and rendered as a distinct badge in src/renderer/src/components/SessionSidebar.tsx (which already renders per-status glyphs), and fires exactly one OS notification per episode reusing the existing Notification pattern. Thresholds and an enable flag are added to the Settings interface + DEFAULT_SETTINGS in src/shared/types.ts and persisted via the existing getSettings/setSettings path; because the app currently has no settings-editing dialog, they ship with sensible defaults (e.g. stall 10 min, unanswered 5 min) and need not have a new settings UI built.

## Specs

- [x] A Claude terminal that stays continuously in the 'working' status longer than the configured stall threshold causes its session row in the sidebar to show a distinct 'stalled' watchdog badge; a terminal that leaves 'working' before the threshold elapses never shows it.
- [x] A Claude terminal that stays continuously in 'needs-attention' longer than the configured unanswered threshold shows a distinct 'unanswered' watchdog badge, visually separate from the normal instantaneous needs-attention glyph.
- [x] Each time a terminal crosses a watchdog threshold, exactly one OS notification is fired for that episode (not one per check tick), and its text identifies the session, the terminal, and which condition (stalled vs unanswered) fired.
- [x] Hovering the watchdog badge shows how long the terminal has been continuously in the abnormal status (e.g. 'working 14m', 'awaiting input 7m').
- [x] When the terminal's status changes away from the watched status (e.g. working→idle, or needs-attention→idle after the user types), the badge clears and the episode resets, so a later re-entry into that status can raise a fresh badge and a new notification.
- [x] Only Claude terminals raise watchdog alerts; plain shell terminals (powershell/cmd/bash/zsh) never trigger a stall or unanswered badge or notification no matter how long they hold a status.
- [x] The stall threshold, the unanswered threshold, and an overall on/off switch are stored in the persisted Settings (with documented defaults) and survive an app restart; setting a given threshold to 0 (or the watchdog to off) suppresses that alert entirely.
- [x] Clicking the watchdog notification focuses and shows the offending session and terminal, matching the behavior of the existing needs-attention notification.
