# Command Palette (Ctrl+K)

A fuzzy-search overlay opened with Ctrl+K (Cmd+K on macOS) that unifies switching and launching: it lists all sessions and worktree tasks (ordered like the sidebar via orderedSessions, each with its live status glyph and, for tasks, the 'branch → baseBranch' label), the saved reusable actions, and files of the active session (its open file tabs, recently changed files from store.recent, and — once the query is 2+ characters — matches found by walking the session folder with the existing window.api.readDir, skipping settings.ignoreNames directories). Implemented as a new renderer component (src/renderer/src/components/CommandPalette.tsx) mounted in App.tsx, with open/close state in the zustand store (src/renderer/src/store.ts); selection delegates to existing store methods setActive, runAction, and openFile. The global keydown handler in App.tsx gains the Ctrl+K binding, and the attachCustomKeyEventHandler bubble-list in src/renderer/src/components/TerminalHost.tsx (the block around lines 134-137 that lets Ctrl+Tab, Ctrl+1-9, Ctrl+B escape xterm) must also let Ctrl+K through so the palette opens even while a terminal has focus, without the keystroke reaching the PTY. Closing the palette restores focus via focusActiveTerminal from termRegistry.

## Specs

- [ ] Pressing Ctrl+K (Cmd+K on macOS) opens the palette overlay with its search input focused, and this works even while an xterm terminal has keyboard focus; the Ctrl+K keystroke never reaches the underlying PTY (no stray character or line-clear appears in the terminal).
- [ ] Pressing Escape, pressing Ctrl+K again, or clicking outside the palette closes it without performing any action, and keyboard focus returns to the active terminal.
- [ ] With an empty query, the palette lists every session and worktree task in the same order as the sidebar, each showing its live status glyph (same glyph mapping as the sidebar) and name; worktree tasks additionally show their 'branch → baseBranch' label; pressing Enter on one switches the app to that session and closes the palette.
- [ ] All saved reusable actions appear in an Actions section; activating one closes the palette and runs the action in the currently active session through the existing runAction flow, focusing the terminal tab the action ran in.
- [ ] Files of the active session appear in a Files section: its open file tabs and recently changed files are listed immediately, and once the query has at least 2 characters, files found under the session folder also match by path — directories named in settings.ignoreNames (node_modules, .git, etc.) are never listed; activating a file opens it in the file viewer tab of the active session and closes the palette.
- [ ] Typing in the search input filters all sections live with case-insensitive subsequence (fuzzy) matching against item labels/paths, sections with no matches are hidden, and a 'no results' message is shown when nothing matches at all.
- [ ] ArrowDown/ArrowUp move the highlighted item across section boundaries and wrap around at the ends; Enter activates the highlighted item; clicking an item with the mouse activates it the same way.
- [ ] All pre-existing shortcuts (Ctrl+Tab, Ctrl+Shift+Tab, Ctrl+1..9, Ctrl+Shift+N, Ctrl+Shift+W, Ctrl+T, Ctrl+Shift+T, Ctrl+B, and Ctrl+F terminal search) continue to work exactly as before when the palette is closed, and while the palette is open none of the typed characters leak into any terminal.
