# Per-Terminal Claude Model Switcher

Add a small model selector to each claude terminal tab that lets the user switch the model the live conversation runs on without manually killing and relaunching. The selector offers Default (no override), Opus, Sonnet, and Haiku. Choosing a model rewrites that terminal's TerminalConfig.claudeArgs so it contains exactly one '--model <alias>' pair (or none for Default), persists it via window.api.updateTerminal -> SessionManager.updateTerminal, and then respawns the PTY in resume mode via store.restartTerminal(terminalId,'resume') so PtySession.spawn() launches 'claude --model <alias> --continue' (PtySession.ts:176-178) — preserving the in-app conversation. The store's restartTerminal bumps epochs so TerminalHost remounts xterm and re-attaches. Key files: src/shared/types.ts (TerminalConfig.claudeArgs), src/main/PtySession.ts (reads claudeArgs + adds --continue), src/main/SessionManager.ts (updateTerminal, restartTerminal, spawnTerminal), src/renderer/src/store.ts (restartTerminal/epochs, updateTerminal wiring), src/renderer/src/components/TabStrip.tsx (where the selector renders on claude tabs). A small pure helper should encapsulate parsing/rewriting the --model value within a claudeArgs array (handling both '--model x' and '--model=x' forms, leaving all other args untouched).

## Specs

- [ ] A claude terminal tab renders a model selector showing the model currently encoded in its claudeArgs (e.g. 'Opus' when claudeArgs contains '--model opus'/'--model=opus', otherwise 'Default'); non-claude tabs (powershell/cmd/bash/zsh) render no selector.
- [ ] Choosing a model X (Opus/Sonnet/Haiku) for a terminal results in that terminal's persisted claudeArgs containing exactly one '--model <alias>' pair with the chosen alias, replacing any previously present --model value and leaving every other arg in the array unchanged and in order.
- [ ] Choosing 'Default' removes the --model flag (and its value) entirely from the terminal's persisted claudeArgs, leaving other args intact.
- [ ] After a model change, that terminal's PTY is respawned and its underlying process command line is 'claude' followed by the updated claudeArgs plus '--continue' (i.e. the conversation is continued, not started fresh), and the xterm view remounts and re-attaches rather than staying detached.
- [ ] Selecting the model that is already active for that terminal makes no change: claudeArgs is unchanged and the PTY is not respawned (no flicker / no --continue restart).
- [ ] The chosen model survives an app restart: a restored claude terminal launches with the saved '--model <alias>' in its arguments (verifiable via the running process args), and Default-model terminals launch with no --model flag.
- [ ] Changing one claude terminal's model affects only that terminal — other claude terminals in the same session (and in other sessions) keep their own model selection and are not respawned.
- [ ] The respawn triggered by a model change seeds the terminal's persisted scrollback so prior output is replayed above the divider, exactly as a normal resume restart does (no loss of visible history).
