# Sentinel finding → one-click fix

On each finding card rendered by RunDetails in src/renderer/src/components/SentinelsPanel.tsx, add two actions: 'Fix with Claude' and 'Copy as prompt'. Both compose a focused fix prompt from the finding's title, detail, and (when present) repo-relative file. 'Fix with Claude' enqueues that prompt onto the watched session's existing prompt queue by calling the already-implemented store action useStore().queueAdd(run.sessionId, prompt) — so the prompt is dispatched to that session's claude terminal the moment it next goes idle, via the unchanged SessionManager idle-dispatch path. 'Copy as prompt' writes the same composed text to the clipboard (navigator.clipboard.writeText) without touching the queue. The target session is always the finding's owning session (run.sessionId), never the currently-active session. This reuses SentinelFinding (src/shared/types.ts), the queueAdd plumbing (src/shared/api.ts, src/renderer/src/store.ts), and the existing queue badge on SessionSidebar.tsx — no new IPC channels, git calls, or main-process code are required.

## Specs

- [ ] Each finding shown inside an expanded sentinel run (RunDetails) displays both a 'Fix with Claude' action and a 'Copy as prompt' action, for findings of every severity, alongside the existing severity dot, detail text, and file-open link (which all continue to render and behave as before).
- [ ] Clicking 'Fix with Claude' on a finding adds exactly one item to the prompt queue of that finding's owning session (run.sessionId), and the queued-prompt badge for that session in SessionSidebar increments by exactly one; no other session's queue changes.
- [ ] The composed prompt text incorporates the finding's title and detail, and when the finding has a `file`, references that exact repo-relative path; when the finding has no `file`, the prompt is still composed from title and detail with no dangling or placeholder file reference.
- [ ] 'Copy as prompt' places the identical composed prompt text on the system clipboard and does NOT add anything to any session's queue (the badge count is unchanged), and gives a brief visible confirmation that the text was copied.
- [ ] The enqueued fix prompt is delivered only through the existing prompt-queue dispatch mechanism: when the target session's claude terminal next sits idle it is submitted automatically, and the feature introduces no new path that types into or submits to a terminal directly.
- [ ] Clicking 'Fix with Claude' shows a transient inline confirmation (e.g. a 'Queued' state) on that specific finding so the user can see the action registered, while other findings in the same run show no such confirmation.
- [ ] Triggering either action on one finding composes the prompt from that finding alone and never bundles in sibling findings from the same run or summary text.
